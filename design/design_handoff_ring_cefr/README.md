## 1. BalanceRing — точная спецификация

Всё это уже зашито в `BalanceRing.tsx`. Список — для QA, **не округлять**.

### Геометрия (viewBox `360 × 360`)
| const | value | смысл |
|---|---|---|
| `CX, CY` | `180, 180` | центр |
| `R0` | `47` | радиус ступицы (хаба) |
| `ORBIT` | `135` | радиус центров иконок — **константа** (это и есть фикс) |
| `T0` | `R0 + 11 = 58` | начало трека освоения |
| `T1` | `ORBIT − 26 = 109` | конец трека освоения |
| `BADGE_R` | `ORBIT + 22 = 157` | центр бейджа — на внешнем ободе, не сталкивается |
| `ARC_R` | `ORBIT + 41 = 176` | радиус групповых дуг |
| `step` | `360 / 9 = 40°` | первая иконка сверху (`−90°`), по часовой |
| ширина луча | `9`, круглые концы | трек и заливка одинаковой ширины |
| радиус иконки | `19` (слабейшая `21`) | |
| иконка (бокс) | `20` (слабейшая `22`), stroke `1.9` | |
| дуга группы | ширина `4`, круглые концы, opacity `0.92` | |

- **Трек**: линия `T0→T1` цветом `track`. **Заливка**: линия `T0→fillR(mastery)`
  цветом группы (слабейшая — красный). `fillR(m) = T0 + clamp(m,0,1)·(T1−T0)`.
  При `mastery ≈ 0` заливки нет — но иконка и бейдж всё равно читаемы.

### Бейдж (`remaining`)
- `remaining > 0` → **пилюля** на `BADGE_R`: высота `20`, `rx 10`,
  ширина `max(20, 12 + len·7)`, заливка `#3B68C9` (слабейшая `#CE6A57`),
  обводка `card 2`, текст `12px/700` белый. `>99` → текст `«99+»`.
- `remaining = 0` (всё выучено) → **зелёный кружок** r `10`, фон `#3B9C6E`,
  белая галочка.

### Формы по группам (`shapes`, по умолчанию `true`)
- `words → circle`, `sent → square` (squircle: сторона `r·1.62`, `rx r·0.42`),
  `read → hex` (шестиугольник, описанный радиус `r·1.06`, вершина сверху).
- `shapes={false}` → все круги (различение только цветом).

### Слабейший режим
- индекс минимального `mastery`. Красный: заливка + обводка иконки + ореол
  (форма, `r+6`, stroke `2.2`, opacity `0.4`) + красный бейдж.

### Центр
- белый круг `R0`, поверх кольцо `goldSoft 3` opacity `.9`; число уровня `36px/800`;
  `УРОВЕНЬ` `11px/700`, tracking `.08em`, цвет `sub`.

### Палитра

|---|---|---|---|---|
| ink | `#23262E` | | track | `#EEEAE2` |
| sub | `#71757F` | | green (done) | `#3B9C6E` |

## 2. Маппинг режимов (порядок важен — под дуги групп)

Порядок `words → sent → read`, 9 режимов. **Иконки выровнены**: оба «печатных»
режима используют `keyboard`.

```ts
import BalanceRing, { type RingMode } from "./BalanceRing";

const m = (group, id, label, icon, r /* ModeReadiness */): RingMode => ({
  group, id, label, icon,
  mastery: r.total ? r.mastered / r.total : 0,
  remaining: Math.max(0, r.total - r.mastered),
});

const modes: RingMode[] = [
  m("words", "recognition",     "Узнавание", "eye",      r.recognition),
  m("words", "production",      "Письмо",    "keyboard", r.production),       // было pen
  m("words", "say_word",        "Речь",      "mic",      r.say_word),
  m("words", "listen_word",     "На слух",   "phones",   r.listen_word),
  m("sent",  "sentences",       "Перевод",   "keyboard", r.sentences),        // было chat
  m("sent",  "say_sentence",    "Речь",      "mic",      r.say_sentence),
  m("sent",  "listen_sentence", "На слух",   "phones",   r.listen_sentence),
  m("read",  "read:text",       "Тексты",    "book",     r.text),
  m("read",  "read:dialog",     "Диалоги",   "masks",    r.dialog),
];
```
> `icon`: `eye | keyboard | mic | phones | book | masks`. Слабейший режим
> вычисляется внутри (`min(mastery)`) — отдельно передавать не нужно.

## 3. CefrMeter — спецификация

Самодостаточный (inline-стили). Пропсы:

```ts
interface CefrBand  { id: string; ru: string; levels: number; }
interface CefrState { bandIdx: number; levelInBand: number; pct: number; nextId: string; }

const BANDS: CefrBand[] = [
  { id: "A1.1", ru: "Основы",          levels: 3 },
  { id: "A1.2", ru: "Повседневность",  levels: 3 },
  { id: "A1.3", ru: "Город и быт",     levels: 3 },
  { id: "A2",   ru: "Уверенный старт", levels: 3 },
];

<CefrMeter bands={BANDS} state={{ bandIdx, levelInBand, pct, nextId }} />
```

- `bandIdx` / `levelInBand` — из текущего глобального уровня (1..12) по карте
  уровень→полоса. `globalLevel = Σ(levels до bandIdx) + levelInBand` (рисуется в чипе).
- `pct` — прогресс **по текущей полосе** (0..1). Сейчас в макете 0.37 для A1.1.
  Если хотите, чтобы это был прогресс **текущего уровня** — передавайте его (только
  визуальная семантика, верстка та же). См. примечание ниже.
- Рейка: по одной ячейке на уровень, сгруппированы по полосам. Пройденные —
  зелёная заливка; текущая — зелёная обводка + заливка на `pct`; будущие — трек.
- Акцент зелёный `#1B8E84`; чип уровня — золотой `#C8902B` / фон `#F6EEDC`
  (как в существующем геро-блоке).

## 5. Чек-лист крайних случаев (что проверить)

- [ ] **Уровень 1**, mastery у всех ~0 → иконки на орбите, не слипаются; бейджи
      «23/19» целиком читаемы (см. `reference.html`, левый телефон).
- [ ] **remaining = 0** → зелёная галочка вместо числа (ур.5: `eye`, `book`).
- [ ] **remaining > 99** → «99+» (пилюля шире, текст влезает).
- [ ] **mastery = 1** → заливка доходит до `T1` (под иконку).
- [ ] **Слабейший** — ровно один, красный с ореолом; если несколько с одинаковым
      минимумом, берётся первый по порядку.
- [ ] **shapes=false** → все круги, различение цветом (фолбэк, если формы не нужны).
- [ ] **CEFR**: текущая ячейка с обводкой + частичной заливкой; пройденные полосы
      с галочкой; `до <nextId>` совпадает со следующей полосой.

## 6. Примечания

- **Шрифт** — Golos Text (как в проде). Числа в SVG заданы `fontFamily="Golos Text"`.
- **Тёмная тема** (опц.) — в `BalanceRing.tsx`/`CefrMeter.tsx` заменить только
  `ink / sub / card / line / track` на ваши `light-dark()`-токены; `green / red /
  blue / group hue / gold` и формы оставить как есть, чтобы семантика читалась
  одинаково.
- **`pct` для CEFR** — если решите показывать прогресс текущего уровня вместо
  полосы, поменяйте только источник `pct`; рейка и подписи не меняются.
- **Тап по иконке** = старт режима (через `onPick(id)`); выученные (`remaining=0`)
  не кликаются (можно сделать «повторение» — добавьте обработчик при желании).
